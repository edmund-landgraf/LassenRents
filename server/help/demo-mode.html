Prototype notes

Demo mode guide

The prototype is intentionally mixed: React pages, Express API stubs, static JSON files, static help pages, and browser-stored demo state.

What is real in the demo

  1. Express serves API stubs, PDF contract previews, file-backed inventory, trucks, work orders, and this help section.
  2. React handles the public site, portal, admin backend, and interactive demo workflows.
  3. Static JSON files in public/data seed inventory, truck, and work-order screens.
  4. Browser local storage keeps test login state and submitted quote leads.
  5. Payments, email sending, and accounting export are visible demo states, not production integrations.

For local development, Vite proxies /api and /help to Express so the app and help pages share the same localhost origin.
