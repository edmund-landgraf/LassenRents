Prototype notes

Demo mode guide

The prototype is intentionally mixed: React pages, Express API stubs, static JSON files used as a local database, static help pages, and browser-stored demo state.

What is real in the demo

  1. Express serves API routes, PDF contract previews, file-backed admin modules, and this help section.
  2. React handles the public site, portal, admin backend, and interactive workflows.
  3. Static JSON files in public/data act as temporary local storage for inventory, trucks, work orders, rentals, invoices, and inspections.
  4. Browser local storage keeps test login state and submitted quote leads.
  5. Card processors, outbound email, and accounting export remain visible demo states, not production integrations.

File-backed admin modules

  1. container-inventory.json via inventory create modal and /api/admin/container-inventory-file.
  2. trucks.json via trucks create modal and /api/admin/trucks-file.
  3. work-orders.json via dashboard work-order modal and /api/admin/work-orders-file.
  4. rentals.json via Rentals New rental modal and /api/admin/rentals.
  5. invoices.json via Mark paid with payment note and /api/admin/invoices/:id/mark-paid.
  6. inspections.json via Inspection master-detail checklist and /api/admin/inspections.

For local development, Vite proxies /api and /help to Express so the app and help pages share the same localhost origin. Restart or refresh after JSON writes if a screen still shows cached seed data.
