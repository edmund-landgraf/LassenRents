Admin backend

Admin operations overview

The backend prototype ties together the office work that happens before, during, and after a rental. Several modules now persist through Express into local JSON files.

Primary modules

  1. Leads: review quote submissions and match them to inventory. Convert to rental opens the Rentals page with a prefilled New rental modal.
  2. Customers: view account details, balances, rentals, notes, and contact actions.
  3. Rentals: manage agreement records from a list. Use New rental to capture customer, container, dates, rates, and fees. Saves through POST /api/admin/rentals into public/data/rentals.json.
  4. Inventory, Trucks, and Work orders: maintain units and dispatch capability with file-backed create flows.
  5. Invoices: record payments with Mark paid. Confirm in the modal and optionally add a payment note. Notes and paid status persist in public/data/invoices.json.
  6. Inspection, Dispatch, Activity, Calendar, and Tax Export: keep field and office work visible.

New rental walkthrough

  1. Open Rentals or click New rental from the dashboard.
  2. Fill customer, site, container unit (from available inventory), dates, and rates.
  3. Save rental to append the agreement to the local rentals JSON file.
  4. From Leads, Convert to rental prefills the same modal from the quote lead.

Mark invoice paid

  1. Open Invoices and choose Mark paid on an open invoice.
  2. Add an optional payment note such as check number, ACH confirmation, or cash received.
  3. Confirm mark paid to zero the balance, set status to Paid, and store the note.
  4. Use Undo payment when you need to restore the prior open balance in the demo.

Demo note: login uses the test admin button and stores authentication in browser local storage. File-backed modules write under public/data through the Express API stub.
